# 周报模板（强制，对齐第四周总结规范）

## 完整模板

```markdown
# Week XX 产品实习总结 — YYYY.MM.DD-MM.DD（可选主题）

> 材料来源：《XXX.pdf》或《XX 对话摘要》（YYYY-MM-DD 传入）
> 本周主线：一句话概括。

---

## 💡 个人思考（最高优先级）

### 1. [认知升级标题]
[反思/洞察/纠偏，挖"为什么"，不只描述"做了什么"]

### 2. ...
### 3. ...

- ...

---

## 📚 方法论沉淀

### 方法论 4：...

---

## 📋 工作任务

### 已完成

- [ ] ...
- [ ] ...

| 维度 | 本周认知升级 |
|------|------|
| 技术理解 | ... |

| 架构思维 | ... |

---

**统计：**
- 📅 时间：WX（YYYY-MM-DD 至 MM-DD）
- 🎯 主线：...
- 💡 关键方法论：N 个
- ❓ 待解问题：N 个
```

## 关键约束

- **日期格式**：ISO 格式 `YYYY.MM.DD-MM.DD`（不要用"X月底至Y月中"这类自然语言）
- **篇幅分配**：个人思考 + 方法论 ≈ 80%，工作任务 ≤ 20%
- **问题复盘**：融入每节内部，不单独建章节；每个维度都追问根因
- **章节顺序与 emoji**：💡 → 📚 → 📋 → 🎯 → 🔑，与第四周总结完全一致
- **统计块**：末尾必须带 `**统计：**` + 📅/🎯/💡/❓ 四行
- **方法论对比**：每条方法论必须标注"新增"或"深化已有框架"（对照 `产品方法论/产品思维框架.md`）

## 内部结构参考

### 💡 个人思考 子节建议
1-4 条认知升级，最后一条通常留"未解的问题/自己留的钩子"作为下轮开头。

### 📚 方法论沉淀 子节建议
编号 N 条，每条至少包含：
- 标题
- 比喻或分类表（比如 `| 组件 | 比喻 | 职责 |` 三列表）
- 1-2 个案例支撑（来自本周实战）

### 📋 工作任务 子节建议
三块固定：**已完成 / 进行中 / 问题清单**（分别用 ✅ / 🔄 / 无前缀）。

### 🔑 核心收获 子节建议
表格两列 `维度 | 本周认知升级`，维度常见：技术理解 / 产品设计 / 架构思维 / 工程实践 / 认知纠偏。
