## 模式识别规则（Step 0 用）

两种模式对应两种工作流分叉：

- **模式 A — 单文档提炼**：读材料 → 提炼 → 只更方法论/memory/已读日志；不生成周报文件、不更新每周索引
- **模式 B — 周总结**：走全 6 步，额外生成 `第X周总结.md` + 更新每周索引

### 识别信号

| 信号 | 判定 |
|------|------|
| 含强触发词：`周报` / `本周总结` / `周总结` / `Week X` / `汇总这周` / `做周报` | **模式 B**（直接进） |
| 单份材料，无周报词 | **模式 A**（直接进） |
| 多份材料（≥2 份 PDF/工单） | **模糊 → 主动问** |
| 单份材料但带"这周/这几天"等时间词 | **模糊 → 主动问** |

### 主动问的话术

> 这份材料你希望走哪种流程？
> A. 单独提炼（只更方法论/memory/已读日志）
> B. 并入本周周报（生成/追加第 X 周总结）

### 已读文档日志"对应总结"列的填法

- **模式 A**：填方法论链接（如 `产品方法论\产品思维框架.md#方法论标题`，日志里用项目根相对路径方便阅读）；无新方法论时填 `N/A`
- **模式 B**：填周报链接（如 `每周总结\第X周总结.md`，同上用相对路径）

- 方法论必须有**具体案例**支撑，不能只是抽象概念
- 与已有方法论对比，明确标注 **新增** 或 **深化已有框架**
- 与最近一份周报的风格/深度对齐，不要自创新模板
- 禁止把 `d:\JMM\...` / `C:\Users\lenovo\...` 写进新生成的文件
