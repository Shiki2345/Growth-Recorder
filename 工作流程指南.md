# 📋 周报总结工作流程指南

## 🎯 系统目标
- 每周记录、总结、沉淀个人工作成果
- 优先级：**个人思考 > 方法论 > 工作任务**
- 为后续建立完整的产品知识库和方法论体系

---

## 🔄 标准工作流（4步）

### Step 1️⃣ 准备文档
```
飞书文档 → 导出PDF → 复制全文内容
```
- 包含：周期、具体工作、想法、遇到的问题

### Step 2️⃣ 分享给Claude
```
粘贴文档内容 + 告诉我：
- 文档周期（如"第4周" 或 "2026.04.18-2026.04.25"）
- 文档来源（如"飞书工作总结"）
```

### Step 3️⃣ Claude处理
Claude会自动：
- ✅ 按"个人思考>方法论>工作任务"优先级总结
- ✅ 生成新的周报文件（如 `第二周总结.md`）
- ✅ 更新`已读文档日志.md`
- ✅ 更新`每周日志索引.md`的表格
- ✅ 保存到本地memory系统（追踪已读内容）

### Step 4️⃣ 方法论沉淀
新提炼的方法论自动分类到：
- `产品方法论/` - 可复用框架
- `术语表.md` - 行业术语积累

## 📊 总结维度详解

### 🧠 个人思考（最优先）
**包含内容**：
- 反思：为什么这样做？有什么启发？
- 洞察：发现了什么规律或问题的本质？
- 感悟：学到了什么，改变了什么认知？
- 转变：思维模式的升级

**例子**：
```
"从本周的访问策略设计中意识到，好产品不是功能堆砌，
而是围绕一个清晰的核心价值递进展开。这改变了我之前
'功能越多越好'的认知。"
```

### 🛠️ 方法论（第二优先）
**包含内容**：
- 框架/模型：可复用的思考方式
- 工具/技巧：实用的操作技能
- 最佳实践：行之有效的做法

**例子**：
```
【三问法框架】
What: 要做什么事情？→ 打造什么功能
Why: 为什么要做？  → 解决什么问题、核心价值
How: 如何落实实现？ → 从用户路径到技术架构
```

### 📋 工作任务（第三优先）
**包含内容**：
- 具体任务列表
- 完成度/进度
- 产出物

**呈现方式**：简化表格或要点列表

### ⚠️ 问题复盘（贯穿全过程）
**关键**：强调根本原因而不只是表象
```
| 问题 | 根本原因 | 解决方案 | 预防措施 |
|-----|---------|---------|---------|
| PRD质量不稳定 | 前期逻辑梳理不够 | 用框架模板逐步推导 | 每次PRD前做流程图 |
```
