## 💡 个人思考（最高优先级）

### 1. 对 Skill 复用机制的认知纠偏

之前一直模糊地以为"每轮对话都会在 skill 里新建模板"，这次亲手跑通 demo2 才把这个概念彻底理清：

- ❌ 不是每轮对话都新建
- ✅ **一轮对话 = 生成 1 个技能**
- ✅ **多轮对话 = 优化同 1 个技能**
- ✅ `skill.zip` 永远只有 1 套技能模板
- ✅ 用户调用时也只会命中这一个

> 这个细节一开始没意识到，容易把"对话历史"和"技能制品"搞混。Skill 是**静态制品**，对话是**动态炼制过程**——炼制过程再长，最终固化下来的只有一套可重入的模板。类似"产品迭代 ≠ 每次都新造一个产品"。

### 2. RAG 是为了防止 AI 幻觉，而不是炫技

demo1 能用多轮对话生成完整 JSON 策略，看起来已经够用。但当我在 demo1 基础上追问"如果用户只说功能名（如'账号管理页面'）而不说具体 URL，能否自动匹配？"——这个问题把单纯 LLM 的天花板暴露了：

- 让 LLM 凭空说 URL，它会编（幻觉）
- 让 LLM 每次都问用户要 URL，产品体验就垮了

**RAG 的真正价值**：把"准确性"这件事从 LLM 的记忆里剥离，交给外挂知识库。LLM 只负责"理解意图 + 发起检索 + 组织回答"，URL 这种**不能错一个字符**的信息必须来自可信源。

### 3. demo1 → demo2 的价值跃迁，本质是"把哪些事固化下来"

效率对比数据（验证同一需求复用 skill）：

| 指标 | 第一次炼制 | 第二次复用 | 变化 |
|------|------|------|------|
| 对话轮次 | 6 轮 | 2 轮 | ↓67% |
| 耗时 | ~15 分钟 | ~2 分钟 | ↓87% |
| 手动决策 | 5 个 | 1 个 | ↓80% |
| 自动化程度 | 20% | 90% | ↑350% |

> 表面看是效率提升，本质是把"决策过程"沉淀成了"决策模板"。第一次用户要回答"审批人选谁/审批模式/权限时长/…"5 个问题，第二次只需要回答"要不要直接复用"1 个问题。**产品的复利正是来自这种结构化固化**——不是让用户每次都重新思考，而是让他们复用已被验证过的最佳路径。

### 4. 未解的问题（自己留的钩子）

- **测试环境怎么选？** 本地跑通 ≠ 可部署。要结合工单里用户的真实场景找一个能对齐的环境。
- **用户表述极度模糊/矛盾怎么兜底？** 目前方案是"多轮对话兜底 + 要求定制"，但"多轮对话"什么时候触发、触发多少轮合适，需要量化标准。
- **知识库手动收集的成本**：没匹配到的页面要用户手动添加+审核入库，这个闭环如果跑不通，RAG 就退化成"半自动百科"。
- **部署复杂度**：demo2 要跑大量 python 脚本，对终端用户/业务侧部署不友好，需要封装。

## 📚 方法论沉淀

### 方法论 1：RAG 与 Skill 的分工

| 组件 | 形象比喻 | 解决什么 | 不解决什么 |
|------|---------|---------|-----------|
| **RAG** | 外挂知识库 | "LLM 不知道具体信息"的问题（URL、业务规则、页面清单等） | 流程怎么跑 |
| **Skill** | 说明书/工具包 | "LLM 不知道怎么一步步办事"的问题 | 具体知识从哪来 |

**组合口径**：RAG 管"信息准确性"，Skill 管"流程可复用性"。两者同时在场，才是"即时可用的策略"。

### 方法论 2：AI 辅助配置类产品的五层架构

源自 demo2 的工程实践：

```
用户输入（自然语言）
   ↓
① 对话引导层（LLM + RAG）——理解意图、多轮追问、调用知识库
   ↓
② 信息收集与验证层——提取关键字段、验证必填项、处理模糊信息
   ↓
③ JSON 生成层——应用默认规则、生成结构化配置、格式校验
   ↓
④ Skill 沉淀层——保存对话模式、提取业务规则、更新知识库
   ↓
⑤ 系统集成层——同步到策略系统、部署到插件、记录执行日志
```

**适用场景**：任何"用自然语言描述 → 生成结构化配置"的产品（访问策略、工单分派规则、审批流、报表筛选器…）。

### 方法论 3：Skill 炼制 vs Skill 复用的两条路径

demo2 验证了同一输入在系统里的两种处理分支：

**第一次（炼制）**：
1. RAG 知识检索（自动）
2. 多轮对话参数收集（6 轮：访问控制类型→审批人→审批模式→临时权限时长→申请理由→生效时间）
3. JSON 策略生成 + 企业级最佳实践自动补齐
4. Skill 模式沉淀（提取触发关键词、决策规则、最佳实践）
5. RAG 使用统计更新

**第二次（复用）**：
1. 文本向量化 + Skill 库相似度匹配（98% 置信度自动触发）
2. 快速复用确认（展示历史数据让用户决策：直接复用/定制/重来）
3. 配置验证与部署准备（跳过中间对话）

**关键阈值**：相似度 ≥ 80% 触发自动推荐，≥ 95% 推荐"直接复用"。

### 方法论 4：验证一条 AI 链路可行性的四个指标

demo2 给出的评估口径：

| 指标 | 意义 | demo2 数值 |
|------|------|-----------|
| 自然语言理解准确率 | 意图识别得对不对 | 97.5% |
| RAG 精确匹配率 | 知识检索返不返得回正确答案 | 100% |
| 结构化输出质量评分 | JSON 能不能直接用 | 98% |
| Skill 可沉淀性 | 能否一次炼制、重复复用 | ✅ 已验证 |
